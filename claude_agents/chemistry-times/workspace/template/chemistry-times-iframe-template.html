GPT-5.4 縮小版來了！Mini 和 Nano 雙殺，免費仔也能用旗艦

OpenAI 今日推出 GPT-5.4 Mini 與 Nano，把旗艦級推理能力塞進最小最快的封裝，Mini 跑分直追 GPT-5.4 本尊、速度快 2 倍以上，Nano 主打高量低價。

等了 12 天，OpenAI 今天把 GPT-5.4 的平民版端上桌——同時推出 Mini 和 Nano 兩個小尺寸版本，分別定位「高性能編程」和「大量呼叫省荷包」。

GPT-5.4 Mini 的定位很明確：它是目前 GPT-5 系列裡最快的編程模型，跑分在 coding、推理、多模態三個面向都超越前代 GPT-5 Mini，而且速度超過 2 倍。在多項 benchmark 上，它已經逼近 GPT-5.4 本尊的成績——等於你花小錢就能拿到接近旗艦的火力。OpenAI 同步宣布 GPT-5 Thinking Mini 將在 30 天內退役，Mini 的世代交替就在此刻。

Nano 走的是另一條路線：極致成本優化。對那些需要大量 API 呼叫但每次任務不需要最頂級推理的場景——例如分類、摘要、批次處理——Nano 提供了一個「夠用就好」的選擇，定價打到骨折。

另一個值得注意的更新：ChatGPT Enterprise 和 EDU 版本同步上線了全新的模型選擇器，依據不同方案（Plus、Pro、Business、Enterprise、Edu）智慧推薦 Instant、Thinking、Pro 三類模型，還加入了自動切換至思考模式的設定。這代表 OpenAI 正在把模型選擇的複雜度從用戶端移到平台端，讓使用者不需要自己判斷「這個任務該用哪個模型」。

化學觀點

GPT-5.4 Mini 的出現，直接打破了「旗艦才能做正事」的舊框架。ChemistryTimes 的 AI 應用場景——教學對話、內容生成、作文批改——其實大部分不需要完整旗艦的火力，Mini 等級就綽綽有餘。工程團隊應該馬上評估：目前跑在旗艦模型上的哪些任務，可以無痛降級到 Mini，省下來的成本直接拿去擴大 AI 功能覆蓋率。這不是降級，是升級策略。

OpenAI Shrinks GPT-5.4 to Mini and Nano — and the Price Floor Just Dropped

The smallest GPT-5.4 variants ship today, making flagship-grade reasoning accessible at a fraction of the cost.

OpenAI shipped GPT-5.4 Mini and Nano on Monday, completing the model family's expansion from flagship to lightweight.

GPT-5.4 Mini is built for speed and coding performance. It outperforms GPT-5 Mini on coding, reasoning, and multimodal tasks while running more than twice as fast. On several benchmarks, it approaches the full GPT-5.4 — meaning developers can get near-flagship quality at significantly lower cost. OpenAI confirmed that GPT-5 Thinking Mini will be retired as a selectable option within 30 days.

Nano occupies the opposite end: maximum throughput at minimum cost. It's designed for high-volume, lower-complexity tasks — classification, summarization, batch processing — where speed and price matter more than peak reasoning depth.

There's also a quiet infrastructure change worth noting. ChatGPT Enterprise and EDU accounts received a streamlined model picker that presents plan-aware options — Instant, Thinking, and Pro models — along with an Auto-switch setting that selects the reasoning mode for you. OpenAI is abstracting away model selection, moving the complexity from the user to the platform.

The pattern is clear: OpenAI's strategy isn't just about building the most capable model. It's about filling every price-performance tier so that no workload has a reason to leave the ecosystem. Mini handles serious coding work. Nano handles the long tail. The flagship handles everything else. Three models, one API, zero gaps.

The ChemistryTimes Take

Mini changes the economics for us. Most of ChemistryTimes's AI workloads — conversational tutoring, content generation, essay feedback — don't require the full GPT-5.4. Mini's coding and reasoning benchmarks suggest it can handle these tasks at a fraction of the cost. Our engineering team should run a structured evaluation this week: identify which production workloads currently on flagship models can shift to Mini without quality degradation. The cost savings fund the next wave of AI feature development.

AI 七巨頭掏 3.75 億台幣，聯手拯救開源安全

Anthropic、AWS、GitHub、Google、Google DeepMind、Microsoft、OpenAI 七大科技公司今日聯合宣布，向 Linux 基金會捐贈 1,250 萬美元，用於強化開源軟體安全防線。

平常打得你死我活的七家公司，今天居然站在同一邊了——而且站的位置不是模型比分榜，是開源安全。

Linux 基金會今天宣布，Anthropic、AWS、GitHub、Google、Google DeepMind、Microsoft、OpenAI 聯合捐贈了 1,250 萬美元（約 3.75 億台幣），資金將透過 Alpha-Omega 和 OpenSSF（Open Source Security Foundation）兩個安全計畫分配。

為什麼現在要做這件事？因為 AI 工具正在以前所未有的速度自動產生安全漏洞報告，這些報告淹沒了開源專案的維護者。絕大多數開源專案是由志工團隊在維護的，他們本來就沒有足夠的人力和預算來處理安全問題，現在 AI 生成的報告量暴增，讓情況雪上加霜。Alpha-Omega 的共同創辦人 Michael Winser 說得很直接：「透過資助審計和在生態系中直接嵌入安全專家，我們已經證明了精準投資是有效的。」

這筆錢具體會用來做什麼？直接跟開源維護者合作、開發以維護者為中心的 AI 安全輔助工具、打造可持續的策略來幫維護者應對日益升級的安全需求。換句話說，不是叫維護者自己想辦法，而是把專業資源送到他們手上。

化學觀點

這則新聞看起來跟 ChemistryTimes 沒有直接關係，但底層邏輯跟我們息息相關。ChemistryTimes 的技術棧從伺服器到前端，大量依賴開源套件。如果這些套件的安全維護跟不上 AI 時代的攻擊速度，我們的供應鏈安全就會出漏洞。這 1,250 萬美元的投入，間接就是在保護我們每天用的工具。工程團隊應該趁這個機會重新檢視一次依賴鏈的安全掃描流程，確保我們有在追蹤上游的安全更新。

Seven AI Rivals Pool $12.5 Million to Shore Up Open Source Security

Anthropic, AWS, GitHub, Google, DeepMind, Microsoft, and OpenAI fund the Linux Foundation to help maintainers fight AI-generated security noise.

Companies that spend most of their time competing just agreed on something: the open source software they all depend on is dangerously under-defended.

The Linux Foundation announced Monday that seven major tech companies — Anthropic, AWS, GitHub, Google, Google DeepMind, Microsoft, and OpenAI — committed $12.5 million to strengthen open source security. The funding flows through Alpha-Omega and the Open Source Security Foundation (OpenSSF), two security initiatives housed within the Linux Foundation.

The problem they're solving is specific and increasingly urgent. AI-powered vulnerability detection tools now discover security issues far faster than volunteer maintainers can triage and fix them. The volume of automated security reports is overwhelming projects that run on donated time and shoestring budgets. The very AI capabilities these companies sell are, inadvertently, creating a burden that threatens the ecosystem all of them rely on.

Alpha-Omega co-founder Michael Winser framed it plainly: "By funding audits and embedding security experts directly into the ecosystem, we've proven that targeted investment works."

The $12.5 million will fund direct collaboration with open source maintainers, development of AI-assisted security tools designed for maintainers rather than attackers, and sustainable strategies to help volunteer-run projects manage escalating security demands.

What makes this notable isn't the dollar amount — $12.5 million is a rounding error for these companies. It's the cooperation. Anthropic and OpenAI are in an active legal dispute over Pentagon AI contracts. Google and Microsoft compete on every AI benchmark. Yet they signed onto the same check because supply chain risk doesn't respect competitive boundaries. Every one of these companies ships products built on open source dependencies. A vulnerability in a widely used Python package doesn't care which AI model found it.

The ChemistryTimes Take

ChemistryTimes's tech stack runs on open source from top to bottom — Node.js, Go modules, Python packages, frontend frameworks. The security of those dependencies isn't someone else's problem; it's ours. This $12.5 million injection into maintainer support is welcome, but it doesn't replace our own responsibility. Engineering should use this as a prompt to audit our dependency scanning pipeline: are we tracking upstream security advisories in real time? Are we patching critical vulnerabilities within the recommended windows? The AI-generated security report flood that prompted this funding will eventually reach our own open source contributions too.

Preply 打出「反 Duolingo」牌，AI 功能卻悄悄跟上

Preply 的策略很有意思：嘴上喊著「我們不是 AI 派」，手上卻悄悄把 AI 功能塞進產品裡。上週推出的「The Better Duo」行銷活動，主打真人家教與學生之間的情感連結——那種 AI 聊天機器人給不了的默契和幽默感。Preply 品牌長 Sofia Tavares 說得很巧妙：「我們不是反 AI，我們是挺人。」

但同時間，Preply 也上線了三個 AI 驅動的新功能：Lesson Insights（AI 自動生成課後個人化摘要）、Daily Exercises（連結課程內容的每日小練習）、Scenario Practice（情境式口說練習，幫學生在課與課之間建立口說信心）。根據 Preply 2026 年效率研究，94% 的學習者表示流利度有提升，91% 在 24 堂課後對口說有信心。

化學觀點

Preply 這招「打反 AI 旗號，做 AI 功能」根本是教科書級的定位操作。ChemistryTimes 手上有更深的真人家教資源，但 AI 輔助工具的落地速度不能輸。課後個人化摘要和情境練習這兩個功能，我們應該認真評估加入產品路線圖。

Preply Plays Anti-AI While Quietly Shipping AI Features

Preply launched "The Better Duo," a campaign designed to position the company as the human-centered antidote to Duolingo's AI-first pivot. Chief brand officer Sofia Tavares threaded the needle carefully: "We're not anti-AI. We're pro-human."

The campaign highlighted real tutor-student pairs — relationships built on empathy, humor, and cultural exchange that chatbots can't replicate. Data from Preply's 2026 Efficiency Study claims learners progress three times faster than industry expectations, with 94% reporting improved fluency and 91% feeling confident speaking after 24-plus lessons.

But here's the interesting part: alongside the pro-human marketing, Preply quietly shipped three AI-powered features. Lesson Insights generates personalized post-lesson summaries. Daily Exercises creates bite-sized practice tied to lesson content. Scenario Practice offers speaking exercises between sessions. The message is "humans first" — but the product is hybrid.

The ChemistryTimes Take

Preply's positioning is clever: own the "human" narrative while matching competitors on AI features. ChemistryTimes has deeper tutor networks than Preply, but we need to match the AI-assisted layer — post-lesson summaries and scenario practice should be on our product roadmap evaluation list.

Mastercard 自建 AI 模型，用數十億筆交易資料抓詐騙

信用卡巨頭 Mastercard 今天宣布，他們不只是「用 AI」，而是直接自己蓋 AI——正在訓練一個大型表格模型（Large Tabular Model, LTM），跟 ChatGPT 那種文字模型不同，這個模型專門吃結構化數據，訓練資料是數十億筆去除個資的交易紀錄。

應用場景鎖定五大方向：網路安全、忠誠計畫、個人化推薦、投資組合優化、數據分析。早期測試顯示，這個模型在降低誤判率（false positives）方面已經超越業界標準的 ML 技術。Mastercard 跟 NVIDIA、Databricks 合作打造這套系統，未來將透過 API 和工具包讓內部團隊存取。

這個動作的意義在於：金融業不再只是 AI 的「消費者」，開始變成 AI 的「製造者」。

化學觀點

Mastercard 做的事情跟 ChemistryTimes 的自適應學習模型有異曲同工之處：都是用大量行為數據去訓練專屬的 AI 模型。差別在於他們有數十億筆交易，我們有數百萬堂課的學習行為。這個方向值得追蹤。

Mastercard Builds Its Own AI Model — Trained on Billions of Transactions

Mastercard announced Monday it's building a generative AI foundation model trained on billions of anonymized transactions — not a language model, but a large tabular model (LTM) designed to work with structured payment data.

The model targets five areas: cybersecurity, loyalty programs, personalization, portfolio optimization, and data analytics. Early testing shows it outperforms standard industry machine learning on reducing false positives — a critical metric in fraud detection. Mastercard partnered with Nvidia and Databricks to build the system and plans to distribute access through internal APIs and toolkits.

The strategic signal: major financial institutions aren't waiting for AI labs to build models for them. They're training domain-specific models on proprietary data that no general-purpose LLM can replicate.

The ChemistryTimes Take

Mastercard's approach mirrors what ChemistryTimes should consider for adaptive learning: training specialized models on proprietary behavioral data. We have millions of lesson sessions worth of learning patterns. Worth tracking this trend.

GitHub 熱門 GitHub Trending Breaking

GitNexus 爆紅：零伺服器、純瀏覽器的程式碼知識圖譜引擎

今天的 GitHub 趨勢榜出現了一個有趣的東西：GitNexus，一個完全在瀏覽器端運行的程式碼智能引擎，不需要後端伺服器。它的核心賣點是把程式碼庫轉換成知識圖譜（Knowledge Graph），然後內建一個 Graph RAG Agent，讓開發者可以用自然語言問問題來理解程式碼結構。

零伺服器架構意味著資料完全不離開用戶的瀏覽器——對重視程式碼安全的企業來說，這是一個巨大的加分項。不用擔心程式碼外洩到第三方伺服器，也不需要額外的基礎建設。

化學觀點

像 GitNexus 這種零伺服器的程式碼理解工具，對內部程式碼庫的新人 onboarding 很有價值。ChemistryTimes 工程團隊可以試用看看。

GitNexus Hits GitHub Trending: A Browser-Only Code Intelligence Engine

GitNexus, built by abhigyanpatwari, landed on GitHub's trending page with a compelling pitch: turn any codebase into a navigable knowledge graph, entirely client-side. The tool runs in the browser with no backend server, which means source code never leaves the user's machine.

Its standout feature is an integrated Graph RAG Agent that lets developers query their codebase using natural language — asking structural questions about dependencies, call flows, and architecture without manually tracing code paths.

The zero-server approach makes it particularly appealing for enterprises with strict code security requirements. No data leaves the browser, no additional infrastructure needed.

The ChemistryTimes Take

A browser-based code exploration tool like GitNexus could help with onboarding new engineers to ChemistryTimes's codebase. Worth a trial run.

Claude 終於有記憶了，跨對話記住你是誰

每次跟 AI 聊天都要重新自我介紹，這個痛終於要結束了。Anthropic 在三月初正式向所有 Claude 用戶推出記憶功能——Claude 現在可以跨對話記住你的名字、寫作風格、偏好設定、甚至你正在進行的專案。以前每次開新對話都像失憶重來，現在 Claude 會自動帶入之前的上下文。

這個功能直接對標 ChatGPT 的記憶功能（2024 年底對付費用戶開放），補齊了 Claude 在個人化體驗上的最大缺口。差異在於實作方式：Claude 的記憶系統強調用戶對記憶內容的控制權，可以查看、編輯和刪除被記住的內容。

化學觀點

AI 記憶功能對教育場景的價值巨大——想像一個家教 AI 能記住學生的學習進度、弱點、偏好的學習方式。ChemistryTimes 在評估 AI 模型整合方案時，記憶功能的成熟度應該列為關鍵評估維度。

Claude Gets Memory — and Your AI Conversations Finally Have Continuity

Anthropic rolled out memory features to all Claude users in early March, letting the AI retain user preferences, writing styles, project context, and personal details across conversations. Previously, every new Claude session started from zero — no continuity, no accumulated understanding.

The feature directly matches ChatGPT's memory capability, which OpenAI shipped to paid users in late 2024. Claude's implementation emphasizes user control: users can view, edit, and delete what Claude remembers. The differentiation is transparency over personalization depth.

For enterprise applications, cross-conversation memory transforms AI from a stateless tool into something closer to a persistent collaborator — one that accumulates institutional knowledge with every interaction.

The ChemistryTimes Take

Cross-conversation memory is transformative for education. An AI tutor that remembers a student's progress, weaknesses, and preferred learning style is fundamentally different from one that starts fresh each session. Memory maturity should be a key evaluation criterion in our AI model integration roadmap.
